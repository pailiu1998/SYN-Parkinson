## 💡 核心结论

### 1. **Test Set 上的权衡**

**Scenario 6 的优势**:
- ✅ 更高的 **Precision** (+1.4%, p=0.003) → 更少的假阳性
- ✅ 更高的 **Balanced Accuracy** (+0.95%, p=0.0002) → 对不平衡数据更稳健

**Original 的优势**:
- ✅ 更高的 **Recall** (+2.4%, p<0.001, large effect) → 更少的假阴性
- ✅ 略高的 **F1 Score** (+0.63%, p=0.028) → 更平衡的精确率和召回率

**解读**: 
- Scenario 6 倾向于"保守预测"（高 precision，低 recall）
- Original 倾向于"激进预测"（高 recall，低 precision）
- 对于帕金森病诊断，**高 recall 可能更重要**（不漏诊）

---

### 2. **Dev Set 上的一致优势**

Scenario 6 在 Dev Set 上几乎全面优于 Original：
- **AUROC**: +1.08% (***) - 效应量巨大 (d=1.09)
- **Recall**: +1.81% (***)
- **F1 Score**: +1.37% (***)
- **Accuracy**: +1.01% (***)
- **Precision**: +0.98% (***)

**这表明**:
- Scenario 6 在 dev set 上有**过拟合**的迹象
- 或者 dev set 的数据分布与 test set 不同
- 训练时选择的最佳模型可能在 test set 上泛化不佳

---

### 3. **Test vs Dev 的不一致性**

| 指标 | Dev Set (S6优势) | Test Set (S6优势) | 一致性 |
|------|------------------|-------------------|--------|
| F1 | +1.37% (***) | -0.63% (*) | ❌ 相反 |
| AUROC | +1.08% (***) | -0.15% (n.s.) | ❌ 相反 |
| Recall | +1.81% (***) | -2.42% (***) | ❌ 完全相反 |
| Precision | +0.98% (***) | +1.38% (**) | ✅ 一致 |

**警告**: Dev 和 Test 表现的巨大差异提示：
- ⚠️ 可能存在**数据泄漏**或**分布不匹配**
- ⚠️ 模型在 dev set 上的选择可能不可靠
- ⚠️ 需要重新审查数据分割策略
